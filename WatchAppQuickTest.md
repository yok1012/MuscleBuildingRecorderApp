# Apple Watch実機テスト手順（ログなし対応版）

## 問題状況
- Xcodeコンソールにログが表示されない
- 心拍数が0のまま
- コードが実行されているか不明

## 対策実施内容

### 1. UI上にデバッグ情報を表示
- 画面上部に黄色いデバッグ情報エリアを追加
- 以下の情報をリアルタイム表示:
  - `Debug Info`: 現在の状態
  - `Auth`: HealthKit権限状態（0/1/2）
  - `Mgr`: WorkoutManagerの状態

### 2. 心拍数取得を手動でトリガー
- 心拍数表示エリアをタップすると手動で取得試行
- 成功時: 実際の心拍数を表示
- エラー時: 99を表示
- データなし: 88を表示

## テスト方法

### ステップ1: 実機で起動
1. XcodeでApple Watchを接続
2. MuscleBuildingRecorderスキームを選択
3. 実機Apple Watchをターゲットに選択
4. Command + R で実行

### ステップ2: UI上のデバッグ情報を確認

**起動直後の表示:**
```
Debug Info
onAppear
Auth: 2 (または 0/1)
Mgr: Init
```

**「開始」ボタンタップ後:**
```
Debug Info
Starting...
Auth: 2
Mgr: Session created (または Error: ...)
```

### ステップ3: 心拍数エリアをタップ
心拍数表示（0 bpm）をタップすると:
- `Mgr: Manual trigger` → データ取得開始
- `Mgr: Got: 72` → 成功（実際の心拍数）
- `Mgr: Query error` → 失敗（99 bpmが表示）
- `Mgr: No samples` → データなし（88 bpmが表示）

## 権限状態の意味
- `Auth: 0` = 未決定（権限要求が必要）
- `Auth: 1` = 拒否（設定で許可が必要）
- `Auth: 2` = 許可済み（正常）

## エラーメッセージ別の対処

### "No HealthKit!"
- watchOSのバージョンが古い
- デバイスの再起動が必要

### "Auth: 1"（権限拒否）
1. iPhoneでヘルスケアアプリを開く
2. 共有 → アプリとサービス
3. MuscleBuildingRecorder
4. すべてのカテゴリをオン

### "Query error"（99表示）
- Apple Watchが手首に装着されていない
- センサーが正しく動作していない

### "No samples"（88表示）
- 最近の心拍数データがない
- ワークアウトを開始してから試す

## 確認事項チェックリスト

- [ ] UI上にデバッグ情報が表示される
- [ ] 「開始」ボタンで状態が変化する
- [ ] 心拍数エリアタップで反応がある
- [ ] Auth状態が2（許可済み）である
- [ ] エラーメッセージが具体的に表示される

## それでも動かない場合

1. **完全にクリーンビルド:**
   - Command + Shift + K
   - Derived Dataを削除
   - 再ビルド

2. **デバイス側の設定確認:**
   - Apple Watch再起動
   - 手首検出がオン
   - パスコード設定済み

3. **最小限のテスト:**
   - 心拍数タップで88/99が表示されるか
   - これが動作すればコードは実行されている
